# Release v0.1.5 — Audit Service, Airflow 3.1.8, SQLAlchemy 2.0 Unification

**Date:** 2026-03-23

## Highlights

This release introduces a **standalone audit service** for compliance-grade event tracking, upgrades **Airflow to 3.1.8**, and **unifies all services on SQLAlchemy 2.0**.

### Audit Trail Service

A new microservice that provides per-customer audit logging across the entire pipeline:

- **Kafka-driven** — Audit events flow through a dedicated `audit.events` Kafka topic, decoupling audit writes from the hot path.
- **Per-customer isolation** — Each customer gets an auto-provisioned `audit_<customer_guid>` MySQL schema, enabling tenant-scoped retention and querying.
- **PII masking** — Sensitive fields (credentials, keys, tokens) are automatically masked before storage.
- **Airflow lifecycle hooks** — A new `audit_listener` plugin captures DAG run and task instance events (start, success, failure) without modifying existing DAG code.
- **Control plane integration** — All integration CRUD operations now emit audit events.
- **Query API** — `GET /api/v1/audit/{customer_guid}` with filtering by event type, integration ID, and date range.

### Airflow 3.1.8 Upgrade

- Upgraded from Airflow 3.0.6 to 3.1.8, which natively supports **SQLAlchemy 2.0**.
- Eliminated the `SQLAlchemy <2.0` constraint that previously forced the Airflow requirements file to pin `1.4.54`.
- All services now run **SQLAlchemy 2.0.48** — no more version split between Airflow and non-Airflow services.
- Custom Airflow Docker image (`Dockerfile.airflow`) bundles shared packages and plugins.

## Upgrade Notes

1. **New Docker service** — Add `audit-service` to your `docker-compose up` command.
2. **Kafka topic** — The `audit.events` topic is auto-created by `kafka-create-topics.sh`. If running outside Docker, create it manually.
3. **MySQL grants** — The audit service needs `CREATE` privileges to auto-provision per-customer schemas. See `docker/mysql-init.sql` for the required grants.
4. **Airflow image** — Airflow now uses a custom image (`Dockerfile.airflow`). Update your image references if deploying outside Docker Compose.
5. **Lock files** — Regenerated for the new Airflow 3.1.8 dependency tree. Run `uv pip sync requirements-lock.txt` to update.
